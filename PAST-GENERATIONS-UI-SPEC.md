# Past Generations View — Build Spec for Claude Code

Paste into Claude Code in the `lc-companion` repo. Goal: a list of all saved sample answers at `/generate/history`, clickable to open a detail view, with Copy (Plain Text) and Copy (HTML) buttons on each, and a Delete button that hard-deletes.

---

## 1. Decisions locked

1. **Hard delete.** `DELETE /api/generate/sample-answer?id=…` performs `DELETE FROM sample_answers WHERE id = ?`. No soft-delete column, no recovery. Confirm dialog on the client before firing.
2. **Two copy formats.** Plain text and HTML. No markdown. Two separate buttons, each copies immediately to the clipboard and shows a 2-second "Copied" confirmation.
3. **Separate page, not modal.** Detail view lives at `/generate/history/[id]`. Shareable URL, back-button works.
4. **No search / filter / pagination yet.** Ship the list, see how it feels with 20+ rows, add filters if needed.

---

## 2. Routes

### `/generate/history` — list view

Server component. Fetches all `sample_answers` ordered by `created_at DESC`. For each row, show:

- Created date (formatted e.g. "23 Apr 2026")
- Poet (from `past_questions.subject_key` joined via `question_id`)
- Question excerpt — first 80 chars of `past_questions.question_text`, ellipsised
- Grade tier (from `pclm_target` — derive tier label from the P score, or add a `tier` column check if already persisted; look at how the form stores it today)
- Approved badge — green "Approved" if `approved=true`, grey "Pending" if not
- Word count

Each row is a `<Link>` to `/generate/history/[id]`.

Empty state: "No sample answers generated yet." with a button link to `/generate`.

Add a subtle link at the top of `/generate` that reads "View past generations →" pointing to this route.

### `/generate/history/[id]` — detail view

Server component fetches the full `sample_answers` row plus the joined `past_questions` row for question text/year. Pass to a client component for the interactive bits.

Layout (reuse the existing card/border styles from `/generate`):

```
┌────────────────────────────────────────────────┐
│ ← Back to history                              │
├────────────────────────────────────────────────┤
│ Question                                       │
│ 2024 — Heaney uses a deceptively simple…       │  (full text, not truncated)
├────────────────────────────────────────────────┤
│ Poet: Seamus Heaney                            │
│ Grade tier: H1                                 │
│ Generated: 23 Apr 2026 · Sonnet 4.6 · 1412 w   │
│ Status: Approved / Pending review              │
│ PCLM target: P 28 · C 27 · L 27 · M 9          │
│ Selected poems: The Forge, Bogland, …          │
├────────────────────────────────────────────────┤
│ Answer                                         │
│ [full prose scroll block, no truncation]       │
├────────────────────────────────────────────────┤
│ [ Copy plain text ]  [ Copy HTML ]             │
│ [ Delete ]                                     │
└────────────────────────────────────────────────┘
```

Keep the existing Approve button if `approved=false`. Don't build a separate edit flow.

## 3. Copy format

Both formats should include the same content in the same order. Only the encoding differs.

**Section order:**

1. Question (year + full text)
2. Grade tier + PCLM target
3. Selected poems
4. Answer text (paragraphs preserved)
5. Generation metadata (model, date, word count)

### Plain text

```
QUESTION
2024, Higher Level Paper 2

Heaney uses a deceptively simple style to convey profound observations about people, places and lives. Discuss this view with reference to the poetry of Seamus Heaney on your course.

GRADE TIER: H1
PCLM TARGET: P 28 · C 27 · L 27 · M 9
SELECTED POEMS: The Forge, Bogland, The Tollund Man, A Constable Calls

ANSWER

There is a temptation, when reading Heaney, to take the surface at face value...

[full answer text with blank lines between paragraphs]

---
Generated by LC Companion on 23 Apr 2026 · Claude Sonnet 4.6 · 1412 words
```

Use `\n\n` between paragraphs. Single blank line between sections.

### HTML

```html
<article>
  <header>
    <h2>Question</h2>
    <p><strong>2024, Higher Level Paper 2</strong></p>
    <p>Heaney uses a deceptively simple style...</p>
  </header>
  <section>
    <p><strong>Grade tier:</strong> H1</p>
    <p><strong>PCLM target:</strong> P 28 · C 27 · L 27 · M 9</p>
    <p><strong>Selected poems:</strong> The Forge, Bogland, The Tollund Man, A Constable Calls</p>
  </section>
  <section>
    <h2>Answer</h2>
    <p>There is a temptation, when reading Heaney...</p>
    <p>[next paragraph]</p>
    ...
  </section>
  <footer>
    <hr>
    <p><small>Generated by LC Companion on 23 Apr 2026 · Claude Sonnet 4.6 · 1412 words</small></p>
  </footer>
</article>
```

Wrap each paragraph of the answer in its own `<p>`. Split on `\n\n`. Escape any `<`, `>`, `&` in the content.

Put both builders in `lib/export/sampleAnswer.ts`:

```ts
export function toPlainText(row: SampleAnswerFull): string { … }
export function toHtml(row: SampleAnswerFull): string { … }
```

Copy via `navigator.clipboard.writeText` (plain) and `navigator.clipboard.write([new ClipboardItem({ 'text/html': ..., 'text/plain': ... })])` (HTML — always include a plain-text fallback in the same ClipboardItem so it pastes correctly into editors that don't accept HTML).

## 4. Delete endpoint

Add `DELETE` handler to `app/api/generate/sample-answer/route.ts`:

```ts
export async function DELETE(request: NextRequest) {
  const id = request.nextUrl.searchParams.get("id");
  if (!id) return NextResponse.json({ error: "missing_id" }, { status: 400 });

  const supabase = getServerSupabase();
  const { error } = await supabase.from("sample_answers").delete().eq("id", id);
  if (error) return NextResponse.json({ error: error.message }, { status: 500 });
  return NextResponse.json({ status: "deleted" });
}
```

Client: show `window.confirm("Delete this sample answer? This cannot be undone.")` before calling. On success, `router.push("/generate/history")`.

## 5. Smoke tests

- [ ] `/generate/history` loads and shows the one Heaney answer you've already generated
- [ ] Clicking a row opens `/generate/history/[id]` with the full answer rendered
- [ ] Copy plain text — paste into TextEdit — shows clean sections with blank-line paragraph breaks
- [ ] Copy HTML — paste into Google Docs — renders with bold labels and paragraph structure
- [ ] Copy HTML — paste into a plain text field (e.g. a Slack message box) — still shows the plain text fallback
- [ ] Delete — confirm dialog appears — confirming removes the row and redirects to `/generate/history`
- [ ] Empty-state copy appears after the only row is deleted

---

## 6. Out of scope (defer)

- Search across question text or poet
- Filter by tier / approved status
- Pagination (not needed until you have 50+ rows)
- Export as .docx or .pdf (current brief is copy only)
- Version compare between two generations of the same question
- Bulk delete
- Undo delete (user explicitly chose hard delete)
